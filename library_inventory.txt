WdfPort.h
  WdfPort (struct) — atomic wave variable port
    Members: Rp (port resistance), a (incident wave), b (reflected wave)
    Methods: voltage(), current(), reset()
    Purpose: Base unit of all WDF elements

LambertW.h
  Namespace: math
  Functions:
    lambertW0(x) — principal branch W0(x) via Halley iteration
    lambertWn1(x) — negative branch W-1(x)
  Purpose: Analytical solution for diode equations

NewtonRaphson.h
  Namespace: math
  Struct: NRResult { value, converged, iterations }
  Functions:
    newtonRaphson(f, df, x0, tol, maxIter) — 1D Newton-Raphson
    halley(f, df, ddf, x0, tol, maxIter) — 3rd-order Halley's method
  Purpose: Nonlinear equation solving for transistors, diodes

WdfOnePort.h
  WdfResistor — pure absorption (b = 0)
  WdfCapacitor — unit delay of incident wave (b = a[n-1])
  WdfInductor — inverted delay (b = -a[n-1])
  WdfIdealVoltageSource — ideal source (b = 2*Vs - a)
  WdfResistiveVoltageSource — voltage source with series resistance
  WdfResistiveCurrentSource — current source with parallel resistance
  Purpose: Linear one-port WDF elements

WdfAdaptors.h
  WdfSeriesAdaptor2<ChildA, ChildB> — 2-port series junction
  WdfParallelAdaptor2<ChildA, ChildB> — 2-port parallel junction
  WdfSeriesAdaptor3<P1, P2> — 3-port series (one port is root)
  WdfParallelAdaptor3<P1, P2> — 3-port parallel
  Purpose: WDF junction elements with scattering relations

WdfNonlinear.h

    Key methods: init(sampleRate, params), reflect(vB, vC, vE), reset()

NOTES
-----

1. All WDF elements (one-ports, adaptors, nonlinear devices) are noexcept
   and use single-precision float (1.0f not 1.0). No heap allocation.

2. Parametric families (WdfDiodeFamily, WdfNpnBjtFamily, WdfPnpBjtFamily)
   are designed as reusable templates that accept Is/Bf/Br/n/Va parameters.
   Preset constants are defined in separate namespace blocks (e.g., diode_params::).

3. Legacy classes (WdfNpnBjt, WdfPnpBjt, WdfIdealDiode, WdfAntiparallelDiodes)
   remain for backwards compatibility but should gradually migrate to parametric
   family classes for new circuits.

4. Op-amp models (LM741, LM308, JRC4558) are NOT WDF elements — they are
   signal processing blocks that take differential input voltage and return
   output voltage. They model slew rate, rail clamping, and frequency response.

5. Two known test failures (2026-03-25):
   - pnp_bjt_test.cpp: suspected Jacobian cross-partial error in WdfPnpBjt
   - wah_test.cpp: suspected pot update not propagating through adaptor tree

6. All effect implementations reference the Patch interface (Patch.h):
   - init(float sampleRate, float* workingBuffer, int workingBufferSize)
   - process(float** inputs, float** outputs, int numSamples)
   - setParameter(int index, float value)  // knobs: 0,1,2; expression: 3
   - buttonPressed(int index)              // 0=left footswitch
   - getLedColor()                          // returns 0xRRGGBB
