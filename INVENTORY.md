## Compiler Constraints & Architecture

### Hard Constraints (All Code)

- **No heap allocation** — no `new`, `malloc`, `std::vector`, `std::string`
- **No exceptions** — compile with `-fno-exceptions`
- **No RTTI** — compile with `-fno-rtti`
- **Single-precision float only** — all literals use `f` suffix (1.0f not 1.0)
- **C++20 standard** — `-std=c++20`
- **noexcept on all audio-path methods** — guarantee no exception throws
- **Position-independent binary** — patches loaded at 0x80000000 (external RAM)
- **Include paths:** `-I. -I./dsp -I./wdf`

### Processor Specifications

| Property | Value |
|----------|-------|
| **CPU** | ARM Cortex-M7 @ 720 MHz |
| **FPU** | FPv5-SP-D16 (single-precision hardware, no double) |
| **Audio** | Stereo, 48 kHz, 24-bit, normalized float [-1.0f, 1.0f] |
| **Working buffer** | 2,400,000 floats in external RAM |
| **DTCM** | For hot-path class members and state |

### Cycle Budget

**Total budget:** ~15,000 cycles/sample for a complete effect

| Element | Cost Range |
|---------|------------|
| Analytical solvers (tanh, sigmoid, Lambert W) | 200–600 cycles |
| 1D Newton-Raphson (diode to ground) | 400–800 cycles |
| 2D Newton-Raphson (BJT Ebers-Moll) | 800–2000 cycles |
| WDF adaptor tree (per node) | 50–100 cycles |
| OnePoleFilter | 30–50 cycles |
| CircularBuffer + Hermite interp | 80–120 cycles |

## Notes

1. **All code is C++20 with noexcept guarantees** on the audio path. The library strictly adheres to hard constraints: no exceptions, no RTTI, no heap allocation during audio processing.

2. **Parametric element families** (WdfDiodeFamily, WdfNpnBjtFamily, WdfPnpBjtFamily, WdfJfetFamily, WdfMosfetFamily) use constexpr preset structs (e.g., `diode_params::`, `bjt_params::`) to enable compile-time device selection without code duplication.

3. **Op-amp models** (LM741, LM308, JRC4558) are NOT WDF elements — they are signal processing blocks modeling slew rate, rail clamping, and frequency response without wave-based scattering.

4. **Two known regressions** (flagged 2026-03-25, targeted for repair):
   - `pnp_bjt_test.cpp`: suspected Jacobian cross-partial error
   - `wah_test.cpp`: suspected pot update not propagating through adaptor tree
   
   Both compile successfully; fix pending.

5. **New DSP primitives** (added 2026-04-02):
   - **BiquadFilter** — 8 types (LP, HP, BP, BS, Peaking, LS, HS, APF)
   - **StateVariableFilter** — VA-modeled SVF with soft saturation
   - **PitchDetector** — AMDF-based with confidence scoring
   - **WindowedSincInterpolator** — 16-tap Blackman-Harris
   - **HaasStereoWidener** — Precedence effect delay modulation
   - **MultiTapDelay** — Tempo-synced multi-tap with feedback
   
   All test harnesses present and passing.

6. **Memory model:**
   - **External RAM (2.4 MB):** Working buffer for delay lines, grain buffers (higher latency)
   - **DTCM:** Hot-path per-sample state (filter coefficients, wave variables, accumulators)
   - **Stack:** Limited — avoid large local arrays in audio callbacks

7. **Effect parameter standardization:**
   - **Knobs:** 3 parameters (indices 0–2), normalized 0.0f–1.0f, always smoothed
   - **Expression pedal:** Parameter index 3, 0.0f–1.0f
   - **Left footswitch:** Tap/hold for mode switching or tap tempo (index 0)
   - **Right footswitch:** Bypass (handled by firmware, not patch code)
   - **LED color:** Returns int 0xRRGGBB from `getLedColor()`
